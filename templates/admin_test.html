Отправить тестовое сообщение

Тестер записывает сообщение от твоего имени напрямую (минуя проверку IP), поэтому его можно использовать с любого устройства, где ты залогинен.

Информация о подключении
IP, привязанный к аккаунту (tablet_ip) {{ tablet_ip or '—' }}
IP браузера, с которого ты сейчас зашёл {{ server_remote_addr }}

MacroDroid / Android-клиент попадёт в сервер с IP, который должен совпадать с tablet_ip, иначе POST /add получит 200 OK, но сообщение не запишется.

Тестирование снаружи (как делает MacroDroid)

Если хочешь проверить именно сетевой путь — отправь POST с другого устройства командой curl или из MacroDroid:

curl -X POST http://<адрес-сервера>:5000/add \
     -d "sender=Иван&text=привет&messenger_name=Telegram"

Этот вариант проходит через привязку по IP, поэтому отправлять надо с того устройства, IP которого записан в tablet_ip выше. Также есть GET /api/ping — простая проверка, что сервер жив.
