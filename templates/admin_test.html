Тестирование сетевого пути

Если хочешь проверить, что сервер реально принимает POST /add снаружи, отправь curl-запрос с Bearer-токеном устройства:

curl -X POST http://<адрес-сервера>:5000/add \
     -H "Authorization: Bearer <токен-устройства>" \
     -d "sender=Иван&text=привет&messenger_name=Telegram"

Без Bearer-заголовка /add работает в IP-fallback режиме: сообщение запишется только если IP отправителя совпадает с tablet_ip выше. Также есть GET /api/ping — простая проверка, что сервер жив.
